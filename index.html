Welcome to my final project for CSE 383: Web Development.

This project utilizes various API's from TomTom in order to give you directions from one place to another via most common ways to travel. There are also options to change what type of route you would like to take to get from point A to point B such as: Fastest, Shortest, Eco-Friendly, and Thrilling! If you pick a thrilling route, you can also adjust how much incline you would like in your route. Once the instructions have been loaded, all of the data related to the request and response are recorded in a database on this ubuntu instance.

If you want to know how this all works, take a look at this diagram!

The system works sequentially through these steps.

  1. Retreive the input data from the form fields Once the button has been pressed.***
  2. Send the starting location and destination query to the fuzzy search API.
    1. Fuzzy search API will return the coordinates of the closest related location to the query string.***
  3. Send the fuzzy results along with the rest of the input data to the routing API.
    1. Routing API will return the total distance, time, delays, and a list of legs with instructions, distance, time, and coordinates.***
    2. For Each leg it will make a JSON object that we add to a list of legs. We also construct a URL for the thumbnail image using the static image map API.***
  4. Now we have all the data, so we display the results of all of this.
  5. Once everything has loaded, we send all data marked with "***" to the database.

*** indicates this step passes the resulting data to the JSON object
